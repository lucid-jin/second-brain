# 🎙️ Voice

이력서·면접 톤의 출발점. **본인이 본인을 어떻게 보는가**.

## Round 1 — 정체성

### Q1. 가장 자랑스러운 작업은?
> 지표 보고 RTC 개선하는 게 뿌듯. AI를 활용해 추측 기반을 확실히 해서 잡는 거.

### Q2. 가장 후회되는 작업은?
> 글쎄… 기억이 안 남. 리뷰 스킬 같은 걸 제대로 안 써본 게 아쉬움. **최선을 나름 다한 것 같음.**

### Q3. 강점/약점 시그니처
> 버그 잡는 걸 좋아함. 측정·진단·픽스 좋아하고, 구조화를 하려고 잘 노력함.

(이전 응답 마지막 "리"는 오타로 정정. 5가지 시그니처 모두 자신 있음.)

## Round 2 — 동기 / 한계

### Q4. AI는 어느 단계에 끼어드나?
> **메트릭 지표를 다양하게 보는 게 가장 큰 장점.** ClickHouse 아니면 잘 안 보였을 것 같음.
> 측정 먼저 → 라이브 나가고 → 코드 만지고 → 검증.
> **혹시 문제 있을 경우 빠르게 돌릴 개선안(rollback plan)도 미리 만들어놓음.**

### Q5. 그 일하는 방식 어디서 배웠나?
(답 누락 — 후속에서 채울 자리)

### Q6. 답답한 것 / 더 하고 싶은데 못 하는 것
> 조금 더 최적화를 위해… **시간이 따로 없는 점**. 라이브러리 버전업 관리. 에러 및 디테일한 부분에 일이 약했던 것 같음.

## Round 3 — 미래 / 회사관

### Q7. 3년 후 어떤 모습?
> **(b) 다른 회사 시니어 IC (RTC/실시간 도메인) + (c) DevEx/플랫폼 팀 리드.**
> 사실 더 큰 풀스택으로도 하고 싶음. **DevOps도 좋음.**

### Q8. 어떤 회사라면 이직 고민?
> **도메인이 중요.**
> (구체 도메인은 후속 인터뷰로 채울 자리)

## 🔍 합성 인사이트

### 1. 진짜 무기는 ClickHouse, AI는 보조 도구
본인이 명확히 자기 인식함: "메트릭 지표 다양하게 보는 게 가장 큰 장점, ClickHouse 아니면 잘 안 보였을 것."
→ 현 이력서 포지셔닝 "AI-Enhanced Frontend"보다 **"Data-Driven Frontend (with AI tooling)"**가 더 정확.
→ Codex 평가에서 "AI-Enhanced 라벨이 baseline화되어 차별점 약화"라고 지적된 부분의 본인 답이 자연스럽게 나옴.

### 2. measure → ship → verify → rollback-ready 사이클
"측정 먼저 → 라이브 나가고 → 코드 만지고 → 검증 → 문제 시 빠르게 돌릴 개선안 미리 준비"
→ dev-N alias 인시던트 1h 8m 클로즈(#2393~5)가 우연이 아니라 **습관화된 사이클**임을 본인 입으로 확인.
→ 시그니처 ⑥(인프라 closed-loop)이 본인 핵심임을 자각.

### 3. 디버거 본질주의 + 자기효능감
"버그 잡는 걸 좋아함" + "후회 없음, 최선 다함."
→ 시니어 시그널: 본인 일에 만족하면서도 자기 도구를 다 안 쓴 미련(코드리뷰 스킬)을 인지.
→ 면접 "왜 이 일 하세요?" 답으로 그대로 사용 가능: **"버그 잡는 게 즐거워서. 데이터로 가설 좁히고 카테고리째 닫는 사이클이 재미."**

### 4. 솔직한 자각 약점
- 최적화 전용 시간 부재 (회사 환경 제약)
- 라이브러리 버전업 관리 체계화 부족 (본인은 axios CVE/livekit upgrade 챙기는 사람인데도 더 체계적으로 하고 싶다는 욕구)
- 에러/디테일 깊이 부족
→ 약점 자가 진단의 정확도가 좋음. 면접에서 "약점은?" 질문에 그대로 사용 가능.

### 5. 확장 의지: 풀스택 + DevOps
"더 큰 풀스택으로도, DevOps도 좋음."
→ 시그니처(시스템 경계 종결, 인프라 closed-loop)와 정확히 일치.
→ 이미 보유: zep-quiz-spring-server 경험, Coolify/Docker/Vercel/Self-hosted GHA 러너 운영.
→ **"Frontend 메인이지만 풀스택/DevOps 확장형 시니어"**로 포지셔닝 가능.

### 6. 도메인 중심 회사 선택
보상/규모보다 도메인. → 채용 컨택이 와도 도메인 fit이 안 맞으면 무시한다는 뜻.
→ 이력서가 도메인 깊이(RTC/실시간/AI 빌더)를 최대한 보여줘야 컨택의 도메인 정확도 올라감.
