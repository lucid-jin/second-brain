# AX 시대, 어떤 도구 조합이 현실적인가

## 핵심 요약
> Obsidian(개인 작업대) + Linear(팀 협업)이 AX 시대에 가장 현실적인 도구 조합이다.

## 배경

Chulhee Lee의 LinkedIn 글 — **"AX의 발목을 잡는 범인은 Notion이다"**를 읽고 토론하면서 정리한 생각.

원글의 핵심 주장:
- 클라우드 도구(Notion, Confluence)는 Agent에게 최적화되지 않았다
- 로컬 plain text 환경에서 Agent가 직접 읽고 쓰는 것과 근본적으로 다른 복잡성이 존재
- Agent에 최적화된 환경을 먼저 구축하고 인간이 적응해야 한다

## 도구별 평가

### Obsidian — 에이전트의 모국어
- 순수 `.md` 파일 → Agent가 Read/Edit/Write로 직접 조작. API 호출 제로
- 속도 최강. 네트워크 레이턴시, 인증, rate limit 전부 없음
- Git 버전 관리 가능
- **한계:** 싱글플레이어. 실시간 협업 없음. 팀 상태 관리 불가

### Linear — 협업 도구 중 가장 agent-friendly
- API 깔끔 (GraphQL 기반, 구조화 잘 됨)
- 마크다운 기반 이슈 본문
- 개발 워크플로우 네이티브 (Git/PR 연동, 상태 전이 명확)
- 비개발자도 쓸 만한 깔끔한 UI (GitHub Projects 대비 훨씬 나음)
- **한계:** 여전히 클라우드 SaaS. 완전한 plain markdown은 아님

### Notion — 비개발자에겐 좋지만 에이전트에겐 고통
- 블록 구조 때문에 API로 쓰기/수정이 복잡 (블록 ID, 타입 파싱 필요)
- 자체 포맷 종속 → 마크다운 변환 시 손실
- 다만 Notion Agents 등 자체 에이전트 기능 확장 중

### GitHub Projects — 아직 미성숙
- ProjectV2 API 지나치게 복잡
- 프로젝트 보드가 Issues 위에 덧붙여진 느낌
- 비개발자 접근성 최하
- Copilot Agent 연동은 가능성 있음

### Jira / Confluence — 레거시
- 무겁고, API 방대하고, 설정 지옥
- 대규모 조직에서 관성으로 쓰이는 것

## 나의 결론

### 최적 조합: Obsidian + Linear

```
Obsidian → 개인 사고 정리, 기술 노트, PRD 초안 (싱글플레이어, 에이전트 최적)
Linear   → 팀 이슈 추적, 상태 관리, 로드맵 (멀티플레이어, agent-friendly)
```

- 에이전트가 **깊은 사고/문서 작업**을 할 때는 Obsidian에서 로컬 마크다운 직접 조작
- 에이전트가 **팀 워크플로우**를 실행할 때는 Linear MCP로 이슈 생성/상태 변경
- 이 두 레이어를 MCP 같은 프로토콜이 연결

### 핵심 인사이트

1. **도구 자체보다 연결 레이어(MCP)가 더 중요하다** — 어떤 도구든 MCP로 연결되면 에이전트가 쓸 수 있음
2. **"로컬 마크다운이 답"은 반만 맞다** — 개인 레이어에서는 맞지만, 팀 협업까지 로컬로 해결하긴 아직 무리
3. **Linear이 균형점을 잘 잡았다** — 에이전트 친화도 + 비개발자 접근성의 트레이드오프가 가장 적음
4. **과도기다** — Nat Friedman의 새 개발자 도구 스타트업($60M 시드), Wrike의 AI Agent 빌더 등이 판을 바꿀 수도 있음

### 주시할 것
- Nat Friedman 스텔스 스타트업 (2026.02, $300M 밸류에이션)
- Wrike 자율 AI Agent / 노코드 에이전트 빌더
- Linear Agentic Backlog 기능 발전
- MCP 프로토콜 생태계 확장
