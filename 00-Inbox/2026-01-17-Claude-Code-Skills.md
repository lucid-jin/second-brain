# Claude Code Skills (스킬)

## 핵심 요약
> 스킬은 Claude에게 특정 작업 수행 방법을 가르치는 마크다운 파일이다. 요청이 스킬과 매칭되면 Claude가 자동으로 사용한다.

## 주요 내용

### 스킬이란?
- Claude Code의 기능을 확장하는 방법
- `SKILL.md` 파일에 지침을 작성하면 Claude가 따름
- **Model-invoked**: 사용자가 호출하지 않아도 자동 적용

### 스킬 저장 위치
| 위치 | 경로 | 범위 |
|------|------|------|
| 개인용 | `~/.claude/skills/` | 모든 프로젝트 |
| 프로젝트용 | `.claude/skills/` | 해당 레포만 |

### 스킬 만드는 법
```bash
mkdir -p ~/.claude/skills/스킬이름
```

```yaml
---
name: 스킬이름
description: 언제 이 스킬을 사용할지 설명
---

# 스킬 제목

여기에 Claude가 따를 지침 작성
```

### 스킬 vs 슬래시 명령어
| 구분 | 스킬 | 슬래시 명령어 |
|------|------|--------------|
| 호출 | 자동 (Claude 판단) | 수동 (`/명령어`) |
| 용도 | 복잡한 워크플로우 | 간단한 반복 작업 |
| 구조 | 디렉토리 + 여러 파일 | 단일 마크다운 |

### 주요 설정 옵션
- `allowed-tools`: 허용할 도구 지정
- `context: fork`: 독립 에이전트로 실행
- `model`: 특정 모델 지정

## 나의 생각

### 이해한 것
- 스킬 = Claude에게 주는 "작업 매뉴얼"
- 한 번 만들어두면 자동으로 적용되니 반복 작업에 좋음
- [[옵시디언]] 정리 스킬처럼 나만의 워크플로우 자동화 가능

### 적용 아이디어
- 코드 리뷰 스킬 만들기
- 커밋 메시지 작성 스킬
- 문서화 스킬 (README 자동 생성)
- 블로그 포스트 작성 스킬

### 의문점
- 여러 스킬이 충돌하면 어떻게 되지?
- 스킬 간 우선순위가 있나?

### 의문점 답변
**Q: 여러 스킬이 충돌하면?**
- Claude가 요청의 맥락을 분석해서 **가장 관련성 높은 스킬 1개만** 선택함
- 충돌은 실제로 거의 일어나지 않음
- 예: "코드 리뷰해줘" → code-review 스킬만, "정리해줘" → obsidian 스킬만

**Q: 스킬 간 우선순위?**
1. 프로젝트 스킬 (`.claude/skills/`) > 개인 스킬 (`~/.claude/skills/`)
2. Description이 구체할수록 정확하게 매칭됨
3. 해결 방법: description을 명확하게 작성 → 자동으로 올바른 스킬 선택
